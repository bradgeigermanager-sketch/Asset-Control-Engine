### Repository scaffold overview
A complete, production‑ready repository scaffold for the **Asset Control Engine** including a Python FastAPI backend, a React + TypeScript frontend, seed data and tests, Docker compose for local development, and CI templates. The scaffold is modular so teams can replace components (Postgres, S3, Vault) with managed services. Below I provide the repository layout, key file contents and snippets, run instructions, and a prioritized checklist mapping to required components.

### Key files and code snippets
**api/main.py** — FastAPI entry with discovery, suggest, and reconcile endpoints
```python
from fastapi import FastAPI, HTTPException
from .matcher import Matcher
from .audit import AuditStore
import json, uuid, datetime

app = FastAPI(title="Asset Control Engine")
with open("api/config.json") as f:
    CONFIG = json.load(f)
matcher = Matcher(CONFIG)
audit = AuditStore("data/audit_store.jsonl")
ASSETS, DISCOVERIES = {}, {}

@app.post("/api/discovery")
def post_discovery(payload: dict):
    did = payload.get("id") or f"d-{uuid.uuid4()}"
    payload["id"] = did
    payload["reconciliationStatus"] = "unmatched"
    DISCOVERIES[did] = payload
    audit.append(actor="system", action="discovery.ingest", resourceId=did, payload=payload)
    return {"id": did}
```

### Setup and run commands
**Local quickstart**
- **Build and run**: `docker-compose -f infra/docker-compose.yml up --build`
- **API**: `http://localhost:8000` OpenAPI at `/docs`
- **UI**: `http://localhost:3000`
- **Generate seeds**: `python data/generate_seeds.py`
- **Run tests**: `pytest -q api/tests/test_reconciliation.py`

**Notes**
- Replace in‑memory stores with Postgres by running `scripts/import_csv.py` to load `assets_seed.csv`.  
- Configure secrets and OAuth in environment variables or Vault before enabling high‑risk endpoints.  
- Use `docker-compose.override.yml` to mount local code for iterative development.

### Implementation milestones and checklist workflow
- **Sprint 0**: repo scaffold, Docker compose, seed generator, OpenAPI contract.  
- **Sprint 1**: implement discovery connectors (AD/MDM), canonical schema, import seeds, basic UI table.  
- **Sprint 2**: matcher core, suggest endpoint, reconciliation apply, audit anchoring.  
- **Sprint 3**: checklist editor, runner UI, evidence upload, approval flows with MFA.  
- **Sprint 4**: workflow orchestrator connectors, sanitization integration, chain‑of‑custody mobile flows.  
- **Sprint 5**: anomaly detection, reporting exports, compliance sign‑offs, production hardening.

**Checklist governance**
- Each checklist item maps to a GitHub issue with owner and acceptance tests.  
- CI gates: unit tests, contract tests, SAST, and reconciliation acceptance thresholds.  
- Security owner signs off before enabling auto‑link or create actions in production.
