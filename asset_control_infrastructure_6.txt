### Implementation notes and recommended extensions

- **MFA scope**: require the `mfa` header for endpoints that perform high‑risk actions (reconcile with create, approve redeploy, vendor changes). Enforce server‑side checks that the MFA token is valid and tied to the approver identity.
- **Audit metadata**: require an `X-Audit-Actor` header or include `actorId` in request bodies for all write operations; every write must be appended to the immutable audit trail with a payload hash.
- **Rate limits and quotas**: add per‑client rate limiting on discovery scan triggers and reconciliation endpoints to prevent abuse.
- **Connector management**: extend the contract with `/api/connectors` to manage connector credentials, rotation, and test runs; store credentials in a secrets manager and never return raw secrets via API.
- **Pagination and filtering**: implement server‑side pagination and indexed filters for `q`, `serialNumber`, and `assetTag` to meet the performance acceptance criteria.
- **Schema evolution**: use `metadata` JSONB for vendor‑specific fields while keeping canonical keys (serialNumber, assetTag, macAddresses) immutable for reconciliation.
